# Sprint 01 — 기획서

**스프린트:** 01 / 기반 레이어  
**기간:** 2026-05-08  
**목표:** 알람 데이터를 저장·로드할 수 있는 기반 레이어 구축

---

## 스프린트 목표

앱의 모든 기능이 의존하는 설정 파일 시스템을 완성한다.  
알람 데이터 구조를 정의하고, JSON 파일로 영속화하는 기능을 검증한다.

---

## 이번 스프린트 범위 (In Scope)

| ID | 기능 | 설명 |
|----|------|------|
| F-07-a | 설정 파일 생성 | 최초 실행 시 `config.json` 자동 생성 |
| F-07-b | 기본값 적용 | 설정 없을 때 안전한 기본값으로 동작 |
| F-06-a | 고정 알람 CRUD | 고정 시각 알람 추가·수정·삭제·조회 |
| F-06-b | 인터벌 알람 CRUD | 인터벌 알람 추가·수정·삭제·조회 |
| F-07-c | 설정 영속성 | 앱 재시작 후에도 설정값 유지 |
| F-07-d | 손상 파일 처리 | JSON 파싱 실패 시 기본값 폴백 |

---

## 범위 외 (Out of Scope)

- UI (메인 창, 트레이, 팝업) — Sprint 03
- 알람 발화 로직 — Sprint 02
- Windows 시작 레지스트리 등록 — Sprint 01에서 구현하되 UI 연동은 Sprint 03
- 빌드/배포 — Sprint 04

---

## 완료 조건 (Definition of Done)

- [o] `config.json` 파일이 `%APPDATA%\TP_alarm\` 에 생성된다
- [o] 고정/인터벌 알람을 추가하면 파일에 저장된다
- [o] 앱 재시작 후 알람 목록이 그대로 복원된다
- [o] 손상된 JSON 파일을 만나도 앱이 크래시 없이 기본값으로 동작한다
- [o] Sprint 01 TC 12개 전부 통과
