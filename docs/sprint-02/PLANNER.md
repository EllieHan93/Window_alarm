# Sprint 02 — 기획서

**스프린트:** 02 / 알람 로직  
**기간:** 2026-05-08  
**목표:** 알람 발화 조건 평가 및 스케줄링 로직 구현

---

## 스프린트 목표

Sprint 01에서 정의한 데이터 모델을 바탕으로, 실제 알람이 올바른 시각과 조건에 발화하는 로직을 완성한다.

---

## 이번 스프린트 범위 (In Scope)

| ID | 기능 | 설명 |
|----|------|------|
| F-01 | 사용 시간 계산 | 앱 시작 / PC 부팅 기준 사용 시간 계산 |
| F-02 | 고정 알람 발화 | 설정 시각 도달 시 알람 이벤트 생성 |
| F-02a | 당일 중복 방지 | 같은 날 동일 알람 1회만 발화 |
| F-02b | 요일 필터 | 지정 요일에만 발화 |
| F-02c | 비활성화 처리 | enabled=False 알람은 발화 안 함 |
| F-03 | 인터벌 알람 발화 | next_fire_epoch 도달 시 발화 후 갱신 |
| F-03a | 재시작 처리 | 앱 재시작 후 epoch 기반으로 정확한 간격 유지 |
| F-99 | 다음 알람 계산 | 전체 알람 중 가장 빠른 발화 시각 계산 |

## 범위 외 (Out of Scope)

- 사용 시간 UI 표시 — Sprint 03

## 완료 조건 (Definition of Done)

- [o] 고정 알람이 설정 시각(±2초)에 큐에 `AlarmEvent`를 put한다
- [o] 당일 이미 발화한 알람은 재발화하지 않는다
- [o] 비활성화 알람은 발화하지 않는다
- [o] 인터벌 알람이 `next_fire_epoch` 경과 후 발화하고 다음 epoch을 갱신한다
- [o] `tick()`이 UI 함수를 직접 호출하지 않는다 (큐에만 put)
- [o] Sprint 02 TC 18개 전부 통과
